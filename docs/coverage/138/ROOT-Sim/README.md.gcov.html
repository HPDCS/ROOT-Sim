      13             : 
      14             : The ROme OpTimistic Simulator is an x86-64 Open Source, distributed multithreaded parallel
      15             : simulation library developed using C/POSIX technology. It transparently supports all the
      16             : mechanisms associated with parallelization and distribution of workload across the nodes
      17             : (e.g., mapping of simulation objects on different kernel instances) and
      18             : optimistic synchronization (e.g., state recoverability).    
      19             : Distributed simulations rely on MPI3. In particular, global synchronization across
      20             : the different nodes relies on asynchronous MPI primitives, for increased efficiency.
      21             : 
      22             : The programming model supported by ROOT-Sim allows the simulation model developer 
      23             : to use a simple application-callback function named `ProcessEvent()` as the event handler,
      24             : whose parameters determine which simulation object is currently taking control for
      25             : processing its next event, and where the state of this object is located in memory. 
      26             : An object is a data structure, whose state can be scattered on dynamically allocated
      27             : memory chunks, hence the memory address passed to the callback locates a top level
      28             : data structure implementing the object state-layout.
      29             : 
      30             : ROOT-Sim's development started as a research project late back in 1987, and is currently
      31             : maintained by the High Performance and Dependable Computing Systems group, a joint
      32             : research group between Sapienza, University of Rome and University of Rome "Tor Vergata".
      33             : 
      34             : ## Installation Notes
      35             : 
      36             : ROOT-Sim uses autotools to provide an installation workflow which is
      37             : common for all supported platforms. This repository does not provide
      38             : already-generated installation scripts (while released tarballs do),
      39             : rather we provide the convenience `autogen.sh` script which should
      40             : build everything on the target machine. Using autotools, `autoconf`,
      41             : `automake` and `libtoolize` are required to let `autogen.sh` generate
      42             : the correct `configure` script.
      43             : 
      44             : Briefly, the shell commands `./configure; make; make install` should
      45             : configure, build, and install this package.
      46             : Also, you can also type `make uninstall` to remove the installed files.
      47             : 
      48             : By default, `make install` installs the package's commands under
      49             : `/usr/local/bin`, include files under `/usr/local/include`, etc.  You
      50             : can specify an installation prefix other than `/usr/local` by giving
      51             : `configure` the option `--prefix=PREFIX`, where `PREFIX` must be an
      52             : absolute path name.
      53             : 
      54             : ROOT-Sim uses many `gcc` extensions, so the currently supported
      55             : compiler is only `gcc`.
      56             : 
      57             : For full configuration notes, and a discussion on the subsystems
      58             : which this simulation library offers, please refer to the 
      59             : [wiki](https://github.com/HPDCS/ROOT-Sim/wiki).
      60             : 
      61             : 
      62             : ## Usage Notes
      63             : 
      64             : When running `make install`, the `rootsim-cc` compiler is added to the path.
      65             : 
      66             : To compile a simulaton model, simply `cd` into the project's directory
      67             : and type `rootsim-cc *.c -o model`. This will create the `model`
      68             : executable, which is the model code already linked with the ROOT-sim library.
      69             : `rootsim-cc` ultimately relies on `gcc`, so any flag supported by
      70             : `gcc` can be passed to `rootsim-cc`.
      71             : 
      72             : To test the correctness of the model, it can be run sequentially, typing    
      73             : `./model --sequential --lp <number of required LPs>`
      74             : This allows to spot errors in the implementation more easily.
      75             : 
      76             : Then, to run it in parallel, type    
      77             : `./model --wt <number of desired threads> --lp <number of required LPs>`
      78             : 
      79             : To run in a distributed environment, you can use standard MPI commands,
      80             : such as:    
      81             : `mpiexec -n 2 --hostfile hosts --map-by node ./model --wt 2 --lp 16`
      82             : 
      83             : This command runs the simulation model on two nodes (`-n 2`) specified in the
      84             : `hosts` file. Each node uses two concurrent threads (`--wt 2`). The simulation
      85             : involves 16 total Logical Processes (`--lp 16`).
