9 ----------------------------------------------------------------------------------------
10 
11 The ROme OpTimistic Simulator is an x86-64 Open Source, distributed multithreaded parallel
12 simulation library developed using C/POSIX technology. It transparently supports all the
13 mechanisms associated with parallelization and distribution of workload across the nodes
14 (e.g., mapping of simulation objects on different kernel instances) and
15 optimistic synchronization (e.g., state recoverability).
16 Distributed simulations rely on MPI3. In particular, global synchronization across
17 the different nodes relies on asynchronous MPI primitives, for increased efficiency.
18 
19 The programming model supported by ROOT-Sim allows the simulation model developer
20 to use a simple application-callback function named `ProcessEvent()` as the event handler,
21 whose parameters determine which simulation object is currently taking control for
22 processing its next event, and where the state of this object is located in memory.
23 An object is a data structure, whose state can be scattered on dynamically allocated
24 memory chunks, hence the memory address passed to the callback locates a top level
25 data structure implementing the object state-layout.
26 
27 ROOT-Sim's development started as a research project late back in 1987, and is currently
28 maintained by the High Performance and Dependable Computing Systems group, a joint
29 research group between Sapienza, University of Rome and University of Rome "Tor Vergata".

31 ## Installation Notes
32 
33 ROOT-Sim uses autotools to provide an installation workflow which is
34 common for all supported platforms. This repository does not provide
35 already-generated installation scripts (while released tarballs do),
36 rather we provide the convenience `autogen.sh` script which should
37 build everything on the target machine. Using autotools, `autoconf`,
38 `automake` and `libtoolize` are required to let `autogen.sh` generate
39 the correct `configure` script.
40 
41 Briefly, the shell commands `./configure; make; make install` should
42 configure, build, and install this package.
43 Also, you can also type `make uninstall` to remove the installed files.
44 
45 By default, `make install` installs the package's commands under
46 `/usr/local/bin`, include files under `/usr/local/include`, etc. You
47 can specify an installation prefix other than `/usr/local` by giving
48 `configure` the option `--prefix=PREFIX`, where `PREFIX` must be an
49 absolute path name.
50 
51 ROOT-Sim uses many `gcc` extensions, so the currently supported
52 compiler is only `gcc`.
53 
54 For full configuration notes, and a discussion on the subsystems
55 which this simulation library offers, please refer to the
56 [wiki](https://github.com/HPDCS/ROOT-Sim/wiki).

59 ## Usage Notes
60 
61 When running `make install`, the `rootsim-cc` compiler is added to the path.
62 
63 To compile a simulaton model, simply `cd` into the project's directory
64 and type `rootsim-cc *.c -o model`. This will create the `model`
65 executable, which is the model code already linked with the ROOT-sim library.
66 `rootsim-cc` ultimately relies on `gcc`, so any flag supported by
67 `gcc` can be passed to `rootsim-cc`.
68 
69 To test the correctness of the model, it can be run sequentially, typing
70 `./model --sequential --lp <number of required LPs>`
71 This allows to spot errors in the implementation more easily.
72 
73 Then, to run it in parallel, type
74 `./model --wt <number of desired threads> --lp <number of required LPs>`
75 
76 To run in a distributed environment, you can use standard MPI commands,
77 such as:
78 `mpiexec -n 2 --hostfile hosts --map-by node ./model --wt 2 --lp 16`
79 
80 This command runs the simulation model on two nodes (`-n 2`) specified in the
81 `hosts` file. Each node uses two concurrent threads (`--wt 2`). The simulation
82 involves 16 total Logical Processes (`--lp 16`).
